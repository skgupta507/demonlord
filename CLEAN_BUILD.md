# Clean Build Instructions

## ⚠️ REQUIRED after updating from a previous version

The errors `Cannot read properties of undefined (reading 'call')` are caused by
a **stale webpack cache** in `.next/` combined with the removed `next-pwa` package.

### You MUST run these commands in order:

```bash
# 1. Delete the stale webpack cache
rd /s /q .next        # Windows
# OR
rm -rf .next          # Mac/Linux

# 2. Delete old node_modules to get clean deps (next-pwa removed)
rd /s /q node_modules # Windows
# OR
rm -rf node_modules   # Mac/Linux

# 3. Reinstall dependencies (next-pwa is now removed)
npm install

# 4. Start dev server
npm run dev
```

### Why this is needed

- `next-pwa@5.6.0` is **not compatible with Next.js 15** and patches webpack internals
  in a way that causes `Cannot read properties of undefined (reading 'call')` errors
  across all Next.js internal modules. It has been removed from this project.
- The `.next` cache still contains compiled chunks that reference the old `next-pwa`
  webpack plugin. Deleting `.next` forces a clean rebuild.
- `GoogleAnalytics` from `@next/third-parties` was placed outside `<body>` which
  caused a React tree error — this has been fixed.
